# AOF v9.6 Release Definition

## Release Theme

Provider Execution Cost and Quota Boundary.

## What This Lets A User Do

An operator can inspect whether a provider-backed action has explicit budget, token, call, retry, quota, rate-limit, billing, and overage boundaries before the action advances.

In practical terms, v9.6 lets AOF stop or escalate provider execution when:

- the budget owner is unclear
- estimated or actual-cost ceilings are missing
- token, call, retry, quota, or rate-limit limits are missing
- billing or overage policy is unclear
- the runtime claims execution eligibility without a bounded budget

## Capability Delta

NEW:

- `PCAP-PROVIDER-COST-QUOTA`: provider execution cost and quota boundary evidence.

UPDATED:

- `PCAP-PROVIDER-EXECUTION`: provider execution candidates now require incident recovery and cost/quota boundaries before moving toward production go/no-go claims.

REMOVED:

- None.

## Contract Surface

v9.6 adds:

- `provider-cost-quota-boundary-record`
- `provider-cost-quota-boundary-audit`
- `aof-provider-cost-quota-boundary-record.schema.json`
- `aof-provider-cost-quota-boundary-audit.schema.json`

The record must bind:

- controlled execution candidate
- execution approval
- incident recovery evidence
- provider scope
- budget owner
- budget period
- estimated cost ceiling
- actual-cost ceiling
- token/call/retry ceilings
- rate-limit boundary
- quota boundary
- billing boundary
- overage policy
- stop conditions
- not-proven boundary

## Required Release Gates

- provider cost quota boundary audit passes
- release-state audit includes provider cost quota boundary audit for v9.6 and later
- provider incident recovery audit remains green
- capability-first release audit remains green
- product value evidence audit remains green
- README, Quickstart, capability matrix, active release manifest, release notes, and release checklist point at v9.6
- full runtime tests and smoke validation pass

## Not Proven

v9.6 does not prove actual provider billing correctness, actual provider quota enforcement, provider semantic correctness, production execution safety, third-party adoption, or market value.

It proves only that AOF can require and audit explicit cost and quota boundaries before provider-backed execution advances.
