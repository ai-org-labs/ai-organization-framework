# Provider Execution Approval Bridge

## Purpose

The approval bridge prevents AOF from treating provider adapter readiness or pilot simulation as permission to create a real external side effect.

## Rule

External provider side effects require a bridge record before execution.

No bridge means no execution claim.

Approved bridge means bounded preflight authorization only. It is not proof that production execution happened or is safe.

## Approval Model

The bridge separates:

- governance boundary readiness
- human approval evidence
- execution scope
- production execution status

`preflight_approved` means the bridge has enough approval evidence to proceed to a later bounded execution step. It does not mean the provider has been called.

## Stop Conditions

Execution must stop if:

- the requested operation changes
- the target provider/resource changes
- human approval is missing or stale
- rollback is not possible
- credentials or budget boundaries are unclear
- a dangerous operation is introduced
- verification evidence cannot be recorded
