# AOF v9.3 Release Definition

## Theme

Capability-First Release Comprehension.

## What v9.3 Must Enable

Users must be able to understand what changed in a release before reading artifact, audit, schema, or dashboard details.

## Required Outcomes

- Define a product capability model separate from the internal actor capability registry.
- Commit a product capability register at `.aof/product-capabilities.json`.
- Add a human-facing capability matrix.
- Add a capability release delta artifact.
- Add `capability-first-release-audit`.
- Require the audit in `release-state-audit` for `v9.3.0` and later.
- Make release notes start with user capability before implementation details.

## Release Acceptance Criteria

- At least one capability is added, improved, or removed.
- Capability delta is recorded.
- One-minute value explanation exists.
- Thirty-second version delta exists.
- README exposes the capability matrix.
- Release notes include `What You Can Do Now`, `Capability Delta`, `Capability Matrix`, and `Value Evidence`.
- Product feedback such as "I do not understand what changed" is treated as a product review trigger.

## Boundary

This release does not prove market adoption, semantic truth, broad user satisfaction, or production provider safety.

It proves that AOF can make release value governable as user-recognizable capability.
