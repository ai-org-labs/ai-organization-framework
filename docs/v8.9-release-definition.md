# AOF v8.9 Release Definition

## Theme

External Runtime Outcome Evidence and Learning Loop.

`v8.8.0` proved that an approved external runtime action can be locally reconstructed and rollback-checked from canonical evidence. `v8.9.0` adds the next gate: AOF must record what outcome was observed and how that outcome changes the learning loop before external runtime claims advance.

## Required Runtime Surface

Required:

- provider outcome evidence records
- provider outcome evidence audit
- provider learning-loop records
- provider learning-loop audit
- Mission Control projection for outcome and learning-loop status
- release-state gates for outcome and learning evidence
- self-hosting `TASK-109` evidence chain

## Acceptance Boundary

This release proves bounded outcome traceability and learning-loop update evidence.

It does not prove:

- production provider execution
- provider semantic correctness
- market truth
- credential or billing safety
- real rollback execution
- autonomous external runtime safety

## Release Decision

Release only if AOF makes it harder to confuse:

- approval with outcome
- reproduction with outcome
- rollback readiness with rollback execution
- observed result with semantic truth
- learning-loop update with market validation
