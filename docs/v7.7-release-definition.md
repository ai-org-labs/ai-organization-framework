# AOF v7.7 Release Definition

## Release

- version: `7.7.0`
- tag: `v7.7.0`
- theme: Adoption-Grade v7 Runtime
- release task: `TASK-096`

## Release Claim

`v7.7.0` turns adoption proof from an onboarding/example claim into a runtime-governed evidence claim.

The bounded claim is:

> AOF can verify that a fresh managed project reaches its first governed work item with work readiness, context integrity, a work execution packet, requirement coverage, provider-neutral session export evidence, release-ready boundaries, and human-readable failure conditions.

## Required Outcomes

- `adoption-proof-benchmark` still verifies the v6.2 worked adoption path.
- `adoption-proof-benchmark` now also produces and verifies AP-007 through AP-011 for the v7 runtime evidence chain.
- AP-007 verifies that a fresh project produces v7 runtime evidence.
- AP-008 verifies readiness and context boundaries.
- AP-009 verifies bounded work execution packet evidence.
- AP-010 verifies requirement coverage evidence.
- AP-011 verifies provider-neutral session export evidence with release-ready boundaries.
- `release-state-audit` includes `adoption-proof-benchmark` for `7.7.0` and later releases.
- TASK-096 has runtime evidence tying the release claim to implementation, governance review, requirement coverage, and session export evidence.

## Authority Boundary

The release does not claim:

- external market adoption
- operator acceptance by a real new customer
- semantic quality of the adopter's product idea
- hosted onboarding automation
- provider-specific managed integrations
- Node.js 25+ support

The release does claim:

- adoption readiness can be tested through a fresh managed-project runtime run
- docs, examples, and artifact count are not adoption proof by themselves
- first governed work can require context, readiness, execution, requirement coverage, and session export evidence
- missing adoption evidence can block release readiness

## Gates

1. Contract gate: adoption proof schema requires AP-001 through AP-011.
2. Runtime gate: `adoption-proof-benchmark` creates v7 runtime evidence in a fresh managed project.
3. Release integration gate: `release-state-audit` includes adoption proof for `7.7.0` and later.
4. Governance gate: TASK-096 has work readiness, context integrity, agent session, Council review, Work Execution Packet, Requirement Coverage, Session Export, and Archmap disposition evidence.
5. Boundary gate: adoption proof is not described as market adoption, semantic truth, operator acceptance, or provider automation.
6. Verification gate: focused runtime tests, governance audits, release-state audit, organization verification, command routing, CLI help benchmark, npm test, and smoke validation pass.
