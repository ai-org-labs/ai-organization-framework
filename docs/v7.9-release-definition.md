# AOF v7.9 Release Definition

## Theme

Externalization Readiness Boundary.

`v7.9.0` is the pre-v8 release that makes externalized organization claims auditable before AOF treats external actors, tools, providers, or references as governed runtime resources.

## Required Outcomes

- Externalized work claims expose source-of-truth, permission, freshness, availability, approval, provenance, and not-proven boundaries.
- `externalization-readiness-audit` fails when those boundaries are missing or unsafe.
- `release-state-audit` includes the externalization readiness gate for `v7.9.0` and later.
- Mission Control and release evidence remain projections from canonical artifacts, not source-of-truth substitutes.

## Boundary

- v7.9 does not execute external providers autonomously.
- v7.9 does not manage secrets, billing, production writes, or deployment.
- v7.9 does not prove semantic correctness of external output.
- v7.9 only proves that externalized runtime claims are structurally and operationally bounded before v8.

## Release Gates

1. `externalization-readiness-audit` passes with at least one canonical externalization claim.
2. `release-state-audit` includes and passes the externalization readiness gate.
3. v7.8 Mission Control projection audit remains green.
4. Existing v7 governance audits remain green.
5. Full tests and smoke validation pass.
