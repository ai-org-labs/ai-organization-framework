# AOF v8.0.0 Release Notes

## Summary

AOF v8.0.0 ships the first Externalized Organization Runtime slice.

External actors, tools, providers, and references can now be represented as
governed runtime resources, and their use can be linked to tasks, sessions,
approval state, outputs, risks, decisions, and release evidence.

## What Changed Since v7.9.0

- Added `external-runtime-resource-record`.
- Added `external-resource-use-record`.
- Added `external-resource-audit`.
- Added schemas for external runtime resources, external resource use, and the
  external resource audit payload.
- Integrated `external-resource-audit` into `release-state-audit` for v8.0 and
  later.
- Added Mission Control external resource projection.
- Added v8.0 self-hosting evidence for `TASK-099` and `TASK-100`.

## Boundary

This release is structural/runtime governance evidence only.

It does not prove autonomous external provider execution, semantic correctness
of external output, credentials/billing/deployment safety, or operator
acceptance of real external workflows.

## Next

Move to provider adapter governance: safe read adapters, approval policy checks,
freshness verification, and explicit escalation before any external-write path.
