# AOF v7.4.0 Release Notes

`v7.4.0` is the Governed Parallel Lanes release.

It follows `v7.3.0` Governed Multi-Actor Pilot. v7.3 made multi-actor claims auditable. v7.4 adds the next missing layer: parallel work must show bounded lanes, lane-local evidence, join/conflict semantics, Council decision, and packet closure.

## What Changed

- Added `parallel-lane-record`.
- Added `parallel-lane-audit`.
- Added schemas for Parallel Lane Pilot and Parallel Lane audit results.
- Integrated `parallel-lane-audit` into `release-state-audit` for `7.4.0` and later releases.
- Added focused runtime tests proving missing lane pilots fail and complete lane pilots pass.
- Added TASK-093 release evidence for readiness, context integrity, session evidence, actor roster, actor handoffs, Council review, Work Execution Packet, Multi-Actor Pilot, Archmap impact, and lane audit.

## Why It Matters

Before v7.4, AOF could show that multiple actors participated, but "parallel" could still mean ungoverned simultaneous activity or disconnected outputs.

After v7.4, AOF can explicitly show:

- which lanes existed
- who owned each lane
- what each lane received as input
- what each lane produced
- what verified each lane
- what blocked or conflicted
- how lanes were joined
- which Council decision authorized merge, stop, defer, or reopen
- what the pilot explicitly does not prove

## Boundary

`v7.4.0` proves structural/runtime parallel lane governance evidence.

It does not prove:

- autonomous scheduling
- speed improvement
- semantic correctness
- provider-native parallel child-agent execution
- external worker adapter readiness
- external adoption

## Post-v7.4 Direction

The next v7.x step should add requirements coverage, forecasting, and organization analytics only after parallel lane join/conflict semantics remain auditable.

Recommended sequence:

1. `v7.5`: Requirements Coverage, Forecasting, And Organization Analytics
2. `v7.6`: Provider-Neutral Session Export
3. `v7.7`: Adoption-Grade v7 Runtime
4. `v8.0`: Externalized Organization Runtime
