# AOF v7.0 Release Definition

## Release

- version: `7.0.0`
- tag: `v7.0.0`
- theme: Agent Session Observability
- parent tasks: `TASK-085`, `TASK-086`, `TASK-087`, `TASK-088`

## Release Claim

`v7.0.0` makes AI organization work reconstructable as runtime session evidence.

The bounded claim is:

> AOF can record, normalize, audit, and project local agent session events so an operator can reconstruct prompt, response, tool call, artifact write, decision, verification, blocker, and stop-condition evidence for implementation-grade work.

## Required Outcomes

- `session-event-record` and `runtime-event-stream` schemas exist.
- `agent-session-record` writes local file-backed session event streams.
- `session-observability-audit` verifies implementation-grade session evidence.
- Agent session records can link task refs, requirement refs, test evidence, risk candidates, decision candidates, and release-ready evidence.
- Council decisions are backed by preceding reviewable evidence events.
- Artifact writes are traceable to producing events.
- Tool calls carry safety level and approval policy metadata.
- Loops can represent goal, verification, and stop-condition events.
- Mission Control can project the latest session and observability audit state from canonical artifacts.
- `mission-control` provides a single operator entrypoint that refreshes current visibility artifacts and starts the viewer.

## Authority Boundary

The release does not claim:

- semantic truth validation
- market or product truth validation
- that a reconstructed session proves the answer is correct
- provider-specific GitHub Copilot, Claude, or Codex streaming integration
- live network streaming
- autonomous multi-actor workforce execution
- Node.js 25+ support

The release does claim:

- session reconstructability can be represented as canonical runtime evidence
- missing session evidence can be treated as a governance blocker
- Mission Control can show current session state without becoming a source of truth
- runtime-backed answers can cite executed command and artifact refs

## Gates

1. Session contract gate: event and stream schemas exist and are covered by tests.
2. Writer gate: `agent-session-record` creates schema-valid local session streams.
3. Audit gate: `session-observability-audit` passes committed v7.0 session evidence and rejects missing or incomplete reconstructability evidence.
4. Projection gate: Mission Control projects latest session, organization, audit, blocker, and next-action state from canonical artifacts.
5. Entrypoint gate: `mission-control` refreshes visibility state and starts Mission Control from one operator command.
6. Governance gate: `TASK-085`, `TASK-086`, `TASK-087`, and `TASK-088` have readiness, review provenance, Archmap impact disposition, and independent verification evidence.
7. Release surface gate: package, project bootstrap, active release manifest, organization release contract, README, checklist, and notes align to `7.0.0`.
8. Verification gate: session observability audit, work readiness audit, quality ledger audit, release-state audit, organization verification, command routing, CLI help benchmark, decision verification, focused tests, full tests, and smoke validation pass.
9. Boundary gate: event streams are not described as semantic truth, implementation quality, or market/product validation.
