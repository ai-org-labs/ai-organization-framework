# AOF v9.4.0 Release Notes

## What You Can Do Now

You can inspect one controlled provider execution candidate instead of manually piecing together many provider-readiness artifacts.

Before v9.4, AOF had approval, target, reproduction, rollback, outcome, learning, operator acceptance, product value, and production-boundary evidence. That was safer than a raw provider write, but still hard to read as a single operator decision.

After v9.4, AOF can say:

> This provider-backed action is a controlled candidate for operator go/no-go. It has upstream evidence, but production execution is still not authorized.

## Capability Delta

NEW:

- None.

UPDATED:

- `PCAP-PROVIDER-EXECUTION`: Provider execution now includes a controlled execution candidate gate above the existing approval/reproduction/rollback/outcome/learning/boundary chain.

REMOVED:

- None.

## Capability Matrix

In thirty seconds:

> v9.4 turns scattered provider readiness evidence into one controlled candidate. The operator can see whether a provider-backed action is ready for go/no-go without mistaking it for production execution.

## Value Evidence

Scenario:

- A user wants to know if AOF can safely approach a provider-backed write.

Before:

- The user had to inspect several artifacts and infer whether they formed a coherent execution candidate.

After:

- The user can inspect a single controlled execution candidate and its audit result.

Reduced work:

- Provider execution readiness no longer requires manually assembling approval, target, rollback, outcome, learning, operator acceptance, product value, and production-boundary artifacts.

Easier judgment:

- The operator can separate "ready for operator go/no-go" from "production execution authorized".

## Release Gate

`release-state-audit` includes `provider-controlled-execution-candidate-audit` for `9.4.0` and later.

## Not Proven

This release does not prove:

- production provider execution safety
- provider semantic correctness
- credential safety in a live provider
- billing safety in a live provider
- actual rollback execution
- market adoption

It proves that AOF can represent and audit a controlled provider-backed execution candidate without authorizing production execution.
