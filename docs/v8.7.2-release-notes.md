# AOF v8.7.2 Release Notes

`v8.7.2` is a patch release for the `v8.7` approval authenticity line.

## What Changed Since v8.7.1

- Reconstructed the release evidence surface for fresh clones.
- Added the committed TASK-107 role result, team output, and council review artifacts that the release-state audit requires.
- Kept the v8.7 approval authenticity boundary unchanged:
  - external write is still not executed;
  - human approval must be explicit, scoped, active, and revocable;
  - provider target payload hash must match the approved scope;
  - credential scope, budget, and rollback remain mandatory before external write authorization.

## Release Boundary

`v8.7.2` does not prove production external execution safety.

It proves that the approval authenticity evidence required by the v8.7 governance contract is reconstructable from the committed repository, including the role work and council review chain used by the release audit.
