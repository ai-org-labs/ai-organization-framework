# AOF v8.8.0 Release Notes

`v8.8.0` ships External Runtime Reproduction and Rollback Proof.

## What Changed Since v8.7

- Added provider execution reproduction records and audit.
- Added provider rollback proof records and audit.
- Added v8.8 release-state gates for reproduction and rollback proof.
- Added self-hosting TASK-108 evidence that reconstructs the approved TASK-107 provider execution preflight path.
- Preserved v8.7 approval authenticity boundaries.

## Operator Meaning

Approval is no longer enough. AOF now requires evidence that the approved external runtime action can be reconstructed and that rollback readiness is auditable before production execution claims are trusted.

## Boundary

This release does not perform production external writes.

It proves local reconstruction and simulated rollback readiness from canonical artifacts only.
