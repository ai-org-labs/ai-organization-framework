# AOF v7.9.0 Release Notes

## Summary

AOF v7.9.0 adds the Externalization Readiness Boundary: a narrow runtime gate for checking whether externalized organization claims are safe enough to become governed runtime resources in v8.

## What Changed

- Added `externalization-readiness-audit`.
- Added the `aof-externalization-readiness-audit` schema.
- Connected the new audit to `release-state-audit` for v7.9 and later.
- Added v7.9 release definition, checklist, and notes.
- Recorded a v7.9 runtime evidence chain for `TASK-098`.

## Why It Matters

v7.8 made Mission Control project coverage and forecast evidence. v7.9 prevents the next step from confusing external tool/provider activity with governed runtime truth. Externalization now requires explicit source-of-truth, permission, freshness, availability, approval, provenance, and not-proven boundaries.

## Boundary

This release does not add autonomous external execution. It prepares the governance boundary required before v8 can externalize organization runtime safely.
