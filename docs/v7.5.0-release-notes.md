# AOF v7.5.0 Release Notes

`v7.5.0` is the Requirements Coverage, Forecasting, And Organization Analytics release.

It follows `v7.4.0` Governed Parallel Lanes. v7.4 made lane split/join claims auditable. v7.5 adds the next missing layer: progress and forecast claims must show which requirements are covered by which work and evidence.

## What Changed

- Added `requirement-coverage-record`.
- Added `requirement-coverage-audit`.
- Added schemas for Requirement Coverage Record and Requirement Coverage audit results.
- Integrated `requirement-coverage-audit` into `release-state-audit` for `7.5.0` and later releases.
- Added focused runtime tests proving missing requirement coverage fails and complete coverage evidence passes.
- Added TASK-094 release evidence for readiness, context integrity, session evidence, actor roster, actor handoffs, Council review, Work Execution Packet, Multi-Actor Pilot, Parallel Lane Pilot, Archmap impact, and requirement coverage audit.

## Why It Matters

Before v7.5, AOF could show governed work happened, but "progress" could still mean task activity rather than requirement coverage.

After v7.5, AOF can explicitly show:

- which requirements are in scope
- which type each requirement is
- which source introduced each requirement
- which work item is linked to each requirement
- which evidence supports the coverage claim
- which requirements are covered, partial, blocked, at risk, or unstarted
- what forecast is being made
- what the forecast explicitly does not prove

## Boundary

`v7.5.0` proves structural/runtime requirement coverage and forecast governance evidence.

It does not prove:

- semantic satisfaction
- market truth
- forecast accuracy
- delivery certainty
- external adoption
- provider-neutral session export

## Verification

- Requirement Coverage audit

- Archmap impact audit
- review provenance audit
- evidence independence audit
- release state audit

- full tests
- smoke validation

## Post-v7.5 Direction

The next v7.x step should package agent/session evidence in a provider-neutral export format only after requirement coverage and forecast boundaries remain auditable.

Recommended sequence:

1. `v7.6`: Provider-Neutral Session Export
2. `v7.7`: Adoption-Grade v7 Runtime
3. `v7.8`: Mission Control Coverage Forecast Projection
4. `v8.0`: Externalized Organization Runtime
