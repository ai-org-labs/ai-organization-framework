# AOF v7.1.0 Release Notes

`v7.1.0` is the Context and Reference Integrity release.

It follows `v7.0.0` Agent Session Observability. v7.0 made AI work reconstructable as session evidence. v7.1 adds the next missing layer: the runtime can now ask whether the session depended on declared, resolvable, fresh-enough context and references.

## What Changed

- Added `context-integrity-record`.
- Added `external-reference-integrity-record`.
- Added `context-reference-integrity-audit`.
- Added schemas for context integrity records, external reference integrity records, and context/reference integrity audit results.
- Integrated `context-reference-integrity-audit` into `release-state-audit` for `7.1.0` and later releases.
- Added Mission Control payload projection for context/reference integrity.
- Added focused runtime tests proving missing context fails and complete references pass.
- Added TASK-090 release evidence for readiness, session, context, external reference, Archmap impact, Council review, and independent verification.

## Why It Matters

Before v7.1, AOF could show what an AI session did, but a human still had to infer whether the session had read the right context or relied on stale external material.

After v7.1, AOF can explicitly show:

- which context packs were used
- which context refs were declared or required
- which refs were missing
- whether hidden context signals exist
- which external references were used
- whether those references were current, stale, available, or unavailable
- what the record explicitly does not prove

## Boundary

`v7.1.0` proves structural/runtime context and reference integrity.

It does not prove:

- semantic correctness
- market value
- product truth
- QIF semantic suitability for every AOF use
- live external reference synchronization
- provider-native session streaming
