# AOF v8.4 Release Checklist

## Gate 1: Externalized Runtime Safety Visibility

- [x] Mission Control renders externalized runtime safety as first-class
  operator evidence.
- [x] Safety states are schema-visible: `safe`, `blocked`, `needs_approval`,
  `stale`, and `not_proven`.
- [x] Safety state links to release, work item, provider/resource, governance,
  and evidence refs.

## Gate 2: Runtime Evidence

- [x] TASK-104 records role results, team output, Council review, and release
  evidence.
- [x] Mission Control explains why externalized runtime work is safe to review
  or blocked.
- [x] Provider-neutral session export remains reconstructable.

## Gate 3: Governance Boundary

- [x] Missing safety evidence is represented as incomplete.
- [x] Permission, approval, risk, freshness, and not-proven boundaries are
  rendered without raw JSON reading.
- [x] Externalized runtime safety evidence is not treated as semantic
  correctness or production-write authority.

## Gate 4: Release Surface

- [x] package version is `8.4.0`.
- [x] active release manifest points at v8.4 release docs.
- [x] roadmap advances the next frontier to v8.5.

## Gate 5: Verification

- [x] `node ./src/cli.js visibility-export --project .`
- [x] `node --test test/runtime-situation.test.js`
- [x] `node --test test/runtime-core-2.test.js`
- [x] `node ./src/cli.js organization-verify --project .`
- [x] `node ./src/cli.js release-state-audit --project . --write-artifact .aof/context/active/release-state-audit.json`
- [x] `npm test`
- [x] `npm run smoke`

## Release Boundary

`v8.4.0` is release-ready only if final verification is green and the release
claim remains bounded to visible externalized runtime safety, not autonomous
external execution.
