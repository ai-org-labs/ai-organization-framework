# AOF v8.0 Direction Review

## Runtime Context

- active release: `v7.9.0`
- frontier task: `TASK-099`
- parent session: `SESS-V80-DIRECTION`
- operating goal: external actors, tools, providers, and references become governed runtime resources under v7.9 readiness boundaries

## Direction Decision

`v8.0` should implement the first Externalized Organization Runtime slice.

The release should not start with autonomous provider execution. It should start
with the canonical runtime objects and gates that make externalized work
governable before any provider, tool, or external actor is trusted.

## Product Meaning

Human-facing wording:

> AOF v8.0 should let the organization use outside actors, tools, providers, and
> references without losing who owns the truth, who approved the work, what was
> actually executed, and what remains unproven.

Operator-facing promise:

- external work is visible as governed runtime resource use
- external work is linked to task, requirement, session, risk, decision, test,
  and release evidence
- external output is never treated as semantic truth only because it exists
- external writes remain approval-bound

## First Bounded Implementation Slice

Implement an External Resource Contract and runtime writer/audit pair.

Required runtime artifacts:

- external-runtime-resource-record
- external-resource-use-record
- external-resource-audit

Required boundaries:

- source-of-truth boundary
- permission boundary
- freshness boundary
- availability boundary
- approval boundary
- side-effect boundary
- provenance boundary
- not-proven boundary

## Release Gates

`v8.0` is release-ready only when:

- external runtime resources can be recorded through CLI/runtime commands
- use of an external resource can be linked to a governed task and session
- audit fails missing source-of-truth, permission, approval, provenance, or
  not-proven boundaries
- release-state-audit includes the external resource audit for v8.0 and later
- Mission Control can show external resource status from canonical artifacts
- tests and smoke remain green

## Deferred

- autonomous external provider execution
- credentials, billing, deployment, or production write management
- hosted multi-tenant runtime
- semantic correctness of external output
- operator acceptance of real external workflows

## Council Recommendation

Proceed to a bounded implementation task for `v8.0` only after this direction
review is accepted. The first implementation task should be narrow: schema,
writer, audit, release-state integration, Mission Control projection, and
self-hosting evidence for one local external resource example.
