# AOF v9.2 Release Definition

## Release Theme

`v9.2.0` ships the External Runtime Production Boundary Candidate.

This release does not execute real provider operations. It defines and audits the pre-production boundary that must be satisfied before controlled provider execution can even be considered.

## Capability First

After v9.2, an operator can answer:

- What provider execution evidence already exists?
- Which proof is still only pre-production evidence?
- Which credential, budget, revocation, rollback, monitoring, incident, human go/no-go, and product-value comprehension boundaries must exist before production execution can advance?
- Whether AOF is blocking production execution rather than quietly implying it is safe.

## New Runtime Surface

- `provider-production-boundary-record`
- `provider-production-boundary-audit`
- `aof-provider-production-boundary-record.schema.json`
- `aof-provider-production-boundary-audit.schema.json`
- Mission Control `provider_production_boundary_projection`
- `release-state-audit` includes `provider-production-boundary-audit` for `9.2.0` and later

## Mandatory Boundary Fields

Production-boundary candidate evidence must include:

- release/work/Mission Control refs
- provider execution approval ref
- reproduction ref
- rollback proof ref
- outcome evidence ref
- learning-loop ref
- operator acceptance drill ref
- product value evidence ref
- provider scope
- credential boundary
- budget boundary
- revocation boundary
- rollback boundary
- monitoring boundary
- incident boundary
- human go/no-go boundary
- product value comprehension boundary
- stop conditions
- provenance refs
- verification refs
- not-proven boundary

## Authority Boundary

`v9.2.0` must not authorize production execution.

The audit intentionally fails if a production-boundary candidate claims `production_execution_authorized: true` or `go_no_go_status: authorized`.

## Not Proven

v9.2 proves structural/runtime pre-production boundary completeness only.

It does not prove:

- production provider execution safety
- credential safety
- billing safety
- provider semantic correctness
- market value
- real rollback execution
- autonomous external runtime readiness

## Release Gate

Release is allowed only if:

- new provider production boundary tests pass
- `provider-production-boundary-audit` passes
- `product-value-evidence-audit` remains green
- `release-state-audit` includes the v9.2 production-boundary gate
- Mission Control can project the boundary without implying production safety
- no `.DS_Store` file is tracked
