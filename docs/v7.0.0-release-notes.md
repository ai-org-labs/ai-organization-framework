# AOF v7.0.0 Release Notes

`v7.0.0` is the Agent Session Observability release.

It follows `v6.9.0` Executable Pre-Implementation Quality Gates. v6.9 made AOF ask whether a work item is ready before implementation begins. v7.0 adds the next missing layer: after work starts, the runtime can now record and audit how the AI organization actually worked.

## What Changed

- Added canonical session event and runtime event stream contracts.
- Added `agent-session-record` for local file-backed agent session streams.
- Added `session-observability-audit` for reconstructability checks.
- Added session evidence links for tasks, requirements, test evidence, risk candidates, decision candidates, and release-ready evidence.
- Projected latest session observability state into Mission Control.
- Added `mission-control` as the operator-facing entrypoint that refreshes current visibility artifacts and starts the viewer.
- Updated command registry, AI help, CLI reference, quickstart, tests, roadmap, and release plan for the v7.0 surface.

## Why It Matters

Before v7.0, AOF could audit final artifacts and release gates, but a human operator still had to reconstruct the live path manually.

After v7.0, AOF can preserve a step-by-step runtime trace:

- what was asked
- which command or tool ran
- which artifact changed
- which council decision was made
- which verification happened
- what risk or decision candidate appeared
- why the loop stopped or moved forward

This makes runtime-backed answers harder to fake and easier to inspect.

## Boundary

`v7.0.0` proves reconstructability of process evidence.

It does not prove:

- semantic correctness
- market value
- product truth
- visual or UX quality
- autonomous multi-actor workforce execution
- live provider streaming integration

Those remain governed by QIF, Guardian/Council review, human/operator validation, outcome feedback, and later v7.x work.
