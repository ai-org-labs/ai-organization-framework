# AOF v8.6.0 Release Notes

## Summary

`v8.6.0` adds the Provider Execution Approval Bridge.

This is the step after v8.5 provider adapter pilot governance. AOF can now record and audit whether a provider-side-effect plan has bounded approval evidence before execution.

## Added

- `provider-execution-approval-record` runtime artifact.
- `provider-execution-approval-audit` runtime verification command.
- Mission Control `provider_execution_approval_projection`.
- Release-state gate for `8.6.0` and later.
- Tests for approval bridge pass and missing human approval failure.

## Boundary

This release does not claim production provider execution safety.

The bridge proves that approval, scope, rollback, credentials, budget, provenance, verification, denied dangerous operations, and stop conditions are represented before execution. It does not prove provider correctness, credential safety, billing safety, market value, or autonomous external execution.
