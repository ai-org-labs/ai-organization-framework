# AOF v9.0 Release Definition

## Theme

External Runtime Operator Acceptance and Safety Drill.

`v8.9.0` proved that an approved, locally reproduced, rollback-ready external runtime path can produce bounded outcome evidence and a learning-loop update. `v9.0.0` adds the missing operator judgment gate: before controlled provider execution advances, a human operator must be able to inspect the chain and record an explicit accept, stop, rollback, escalate, or defer decision.

## Required Contract

The release introduces:

- `operator-acceptance-drill-record`
- `operator-acceptance-drill-audit`
- Mission Control projection for operator acceptance drill status
- release-state audit gate for `v9.0.0` and later

The drill must link:

- provider execution approval
- provider execution reproduction
- provider rollback proof
- provider outcome evidence
- provider learning-loop update
- Mission Control packet
- operator decision rationale
- accepted risk
- blocker summary
- safety boundary
- not-proven boundary

## Allowed Decisions

- `accept`
- `stop`
- `rollback`
- `escalate`
- `defer`

## Release Claim

AOF can now prove that a bounded external runtime chain has a recorded operator acceptance and safety drill decision before controlled provider execution advances.

## Not Proven

This release does not prove:

- live provider behavior
- production external write safety
- credential safety
- billing safety
- semantic truth
- market truth
- independent third-party operator acceptance
- actual production rollback execution

## Release Gate

Release is valid only if:

- `operator-acceptance-drill-audit` passes
- the drill links approval, reproduction, rollback, outcome, learning, and Mission Control evidence
- `release-state-audit` includes and passes the v9.0 operator acceptance gate
- Mission Control shows the operator acceptance projection
- existing v8.9 outcome and learning-loop gates remain green
