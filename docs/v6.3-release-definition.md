# AOF v6.3 Release Definition

## Release

- version: `6.3.0`
- tag: `v6.3.0`
- theme: AI Command Help Surface with QIF v0.3 Provider Alignment
- parent task: `TASK-066`

## Release Claim

`v6.3.0` makes AOF easier for AI orchestrators to operate correctly by exposing compact command help from the canonical command registry.

The release claim is intentionally bounded:

> An AI can ask AOF which command to use without reading the full CLI reference, and command help states purpose, category, inputs, outputs, failure meaning, and QIF boundary.

## Required Outcomes

- `aof --help` returns a compact AI-oriented command index.
- `aof --help --json` returns a machine-readable command index.
- `aof <command> --help` returns command-level help.
- `aof <command> --help --json` returns structured command help.
- Help is generated from the command registry rather than a separate manual.
- Help includes failure meaning so a pass/fail result is not opaque.
- Help includes QIF mapping or QIF boundary language.
- `cli-help-benchmark` verifies supported command help coverage.
- The active QIF provider profile points to QIF `v0.3.0`.
- QIF v0.3.0 Discovery Layer is recorded as provider-level alignment without overclaiming v0.3 executable verifier support.

## Authority Boundary

The release does not claim:

- that help text proves command semantics are correct
- that command success proves semantic quality truth
- that QIF v0.3.0 provides executable AOF verifier rules
- that every benchmark result is fully human-explained
- that Mission Control UX is solved

The release does claim:

- AI command discovery is now a runtime surface
- command help is structured and registry-backed
- important benchmark and verification commands expose QIF intent/risk/loss-boundary style language
- AOF can track QIF v0.3.0 as an external provider profile while preserving executable baseline honesty
