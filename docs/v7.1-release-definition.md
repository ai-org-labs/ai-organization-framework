# AOF v7.1 Release Definition

## Release

- version: `7.1.0`
- tag: `v7.1.0`
- theme: Context and Reference Integrity
- release task: `TASK-090`

## Release Claim

`v7.1.0` makes AOF release claims harder to fake by requiring implementation-grade work to declare the context and external references it depended on.

The bounded claim is:

> AOF can record, audit, and project context dependencies, external reference freshness, hidden-context signals, and not-proven boundaries before a release is treated as ready.

## Required Outcomes

- `context-integrity-record` writes schema-valid context dependency records.
- `external-reference-integrity-record` writes schema-valid external reference freshness and availability records.
- `context-reference-integrity-audit` verifies context records, required refs, external refs, blocked context, stale refs, and not-proven boundaries.
- `release-state-audit` includes `context-reference-integrity-audit` for `7.1.0` and later releases.
- Mission Control payload includes context/reference integrity projection from canonical artifacts.
- TASK-090 has work readiness, Archmap impact, Council review, independent evidence, session evidence, and context/reference integrity evidence.

## Authority Boundary

The release does not claim:

- semantic truth of referenced material
- market/product truth
- that QIF or any external reference is correct for every use case
- live external-reference synchronization
- autonomous provider session streaming
- Node.js 25+ support

The release does claim:

- missing or stale context can be made visible before release sign-off
- external references can be represented with freshness and availability boundaries
- not-proven limits are mandatory on context/reference integrity records
- context/reference integrity can become a release-state governance gate

## Gates

1. Context contract gate: context and external-reference integrity schemas exist.
2. Writer gate: both writer commands create schema-valid artifacts.
3. Audit gate: `context-reference-integrity-audit` fails missing context and passes TASK-090 release evidence.
4. Release integration gate: `release-state-audit` includes context/reference integrity for `7.1.0` and later.
5. Mission Control gate: visibility export projects context/reference integrity status from canonical artifacts.
6. Governance gate: TASK-090 has work readiness, Archmap impact, Council review provenance, and independent evidence.
7. Boundary gate: context/reference integrity is not described as semantic truth.
8. Verification gate: focused runtime tests, governance audits, release-state audit, organization verification, command routing, CLI help benchmark, npm test, and smoke validation pass.
