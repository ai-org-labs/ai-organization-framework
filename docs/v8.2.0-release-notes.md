# AOF v8.2.0 Release Notes

## Summary

`v8.2.0` adds the Adoption Feedback And Operator Validation Loop.

The release makes operator/adopter feedback a first-class runtime artifact so
AOF can stop confusing internally green work with operator-accepted work.

## Added

- `operator-validation-record` CLI command.
- `operator-validation-audit` CLI command.
- Operator validation record and audit schemas.
- `release-state-audit` gate for `operator-validation-audit` on `v8.2+`.
- Mission Control `operator_validation_projection`.
- Self-hosting `TASK-102` operator validation evidence.

## Boundary

Operator validation is governed feedback evidence. It is not market truth,
semantic correctness, broad adoption, or product value proof.

## Next

The next frontier is `v8.3`: Mission Control Operator Acceptance Evidence.
