# AOF v9.5 Release Definition

## Release Theme

External Runtime Incident Drill and Recovery Evidence.

## What This Lets A User Do

An operator can inspect what AOF will do if a controlled provider-backed action fails before allowing the action to advance.

In practical terms, v9.5 adds a governed recovery drill:

- detect the incident signal
- stop or contain further provider calls
- decide whether rollback is required and ready
- record the recovery action
- notify the operator with evidence refs
- update the learning loop
- block, defer, escalate, or resume only through explicit governance

## Capability Delta

NEW:

- `PCAP-PROVIDER-INCIDENT-RECOVERY`: provider incident recovery evidence.

UPDATED:

- `PCAP-PROVIDER-EXECUTION`: controlled execution candidates now require incident recovery evidence before the provider track can advance.

REMOVED:

- None.

## Contract Surface

v9.5 adds:

- `provider-incident-recovery-record`
- `provider-incident-recovery-audit`
- `aof-provider-incident-recovery-record.schema.json`
- `aof-provider-incident-recovery-audit.schema.json`

The record must bind:

- controlled execution candidate
- execution approval
- reproduction proof
- rollback proof
- outcome evidence
- learning-loop update
- operator acceptance drill
- production boundary
- incident scenario
- detection signal
- containment action
- rollback decision
- recovery action
- operator notification
- resume/stop governance
- not-proven boundary

## Required Release Gates

- provider incident recovery audit passes
- release-state audit includes provider incident recovery audit for v9.5 and later
- capability-first release audit remains green
- product value evidence audit remains green
- README, Quickstart, capability matrix, active release manifest, release notes, and release checklist point at v9.5
- full runtime tests and smoke validation pass

## Not Proven

v9.5 does not prove that a live provider incident can be recovered in production.

It does not prove provider semantic correctness, credential safety, billing safety, customer impact absence, actual rollback execution, third-party operator adoption, or market value.

It proves only that AOF can require and audit a reconstructable incident recovery drill before the external runtime advances.
