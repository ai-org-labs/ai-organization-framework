# AOF v9.3.0 Release Notes

## What You Can Do Now

You can judge an AOF release by user capability first.

Before v9.3, AOF could pass many runtime audits while a user still had to ask: "What can I do now that I could not do before?"

After v9.3, a release must expose:

- the capability that changed
- the user value of that capability
- when each capability first appeared
- what changed in this version
- what evidence supports the value claim
- what remains unproven

## Capability Delta

NEW:

- `PCAP-CAPABILITY-FIRST-RELEASE`: Release readiness now requires a product capability register, capability matrix, capability delta, and audit.

UPDATED:

- `PCAP-PRODUCT-VALUE-COMPREHENSION`: Product value evidence is no longer only a release artifact; it is connected to a formal capability model and release acceptance gate.

REMOVED:

- None.

## Capability Matrix

See [docs/aof-capability-matrix.md](aof-capability-matrix.md).

In thirty seconds:

> v9.3 turns release value from a written explanation into a governed capability gate. AOF must now show the user-facing capability delta before claiming release readiness.

## Value Evidence

Scenario:

- A user opens the latest release and wants to know whether v9.3 matters.

Before:

- The user had to infer value from records, audits, schemas, gates, and release history.

After:

- The user can read the capability matrix and see that v9.3 added Capability-First Release as a governed release requirement.

Reduced work:

- The user no longer needs an AI-generated version comparison to know what the release claims to improve.

Easier judgment:

- The user can reject a release explanation that starts from mechanism instead of capability.

## Release Gate

`release-state-audit` includes `capability-first-release-audit` for `9.3.0` and later.

## Not Proven

This release does not prove:

- market adoption
- semantic correctness
- broad user satisfaction
- production provider safety
- that every historical release is easy for every user to understand

It proves that AOF can make future release claims capability-first and runtime-auditable.
