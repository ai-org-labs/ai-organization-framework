# AOF v7.8.0 Release Notes

## Summary

`v7.8.0` ships Mission Control Coverage Forecast Projection.

Mission Control now exposes requirement coverage, forecast boundary, adoption proof status, and evidence completeness from canonical runtime artifacts instead of requiring operators to inspect raw JSON manually.

## Added

- Added `requirement_coverage_projection` to Mission Control.
- Added `adoption_proof_projection` to Mission Control.
- Added `evidence_completeness_projection` to Mission Control.
- Added `mission-control-projection-audit`.
- Added `schemas/aof-mission-control-projection-audit.schema.json`.
- Added release-state audit integration for `mission-control-projection-audit` on `7.8.0` and later.

## Changed

- Extended `schemas/aof-mission-control-view.schema.json` to require v7.8 projection fields.
- Updated command catalog and command registry metadata so AI can route to the new audit command.
- Updated release state and project bootstrap to `7.8.0`.
- Updated roadmap and release plan to move the frontier to v8.0 after the v7.8 release.

## Boundary

This release does not claim that Mission Control visual design is final or that forecast estimates prove delivery certainty. It claims that the visibility surface is now backed by explicit runtime evidence projection and can fail release readiness when required projection sources are missing.
