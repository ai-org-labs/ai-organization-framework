# AOF v6.0.0 Release Notes

## Summary

`v6.0.0` is the Public Runtime Readiness release.

It follows the `v5.0.0` Skillful Actor Runtime release and focuses on making the historyless public AOF baseline usable, inspectable, and honestly bounded for first-time adopters.

## What Changed Since v5.0.0

- Added a v6.0 public runtime readiness definition.
- Added v6.0 release definition, checklist, and release notes.
- Updated README and quickstart to point at the public `ai-org-labs` baseline and `v6.0.0`.
- Clarified that examples are starter templates: seed configuration is committed, runtime artifacts are generated during use.
- Clarified that the viewer is a runtime-backed visibility surface, not a finished Human Recognition UX.
- Ran v6.0 direction through AOF runtime and Need Validation before opening project execution.
- Updated package metadata toward `6.0.0`.

## Known Boundaries

- GitHub push/release requires `ai-org-labs` write access. A local release/tag can be prepared before remote publication.
- Current viewer is not a polished one-screen organization UI.
- Live model/provider execution is environment-dependent.
- Validator pass proves structure, references, and runtime rule compliance; it does not prove semantic or market truth.

## Recommended Next Frontier

After v6.0, the next frontier should be selected by runtime review. The likely candidates are:

- v6.1: public onboarding worked example and command readiness matrix expansion
- v6.2: Human Recognition Interface redesign as a true one-screen organization view
- v7.0: governed multi-actor orchestration beyond a single parent Codex runtime
