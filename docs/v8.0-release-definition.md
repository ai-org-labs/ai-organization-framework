# AOF v8.0 Release Definition

## Release Theme

Externalized Organization Runtime.

`v8.0.0` turns external actors, tools, providers, and references into governed
runtime resources. The release does not execute external providers
autonomously; it defines the canonical resource/use contract and audit boundary
required before externalized work can be trusted by AOF.

## Required Outcomes

- `external-runtime-resource-record` writes governed external resource artifacts.
- `external-resource-use-record` links an external resource to task, session,
  operation, approval status, outputs, risks, decisions, provenance, and
  not-proven boundary.
- `external-resource-audit` fails missing source-of-truth, permission,
  freshness, availability, approval, side-effect, provenance, operation, or
  not-proven boundaries.
- `release-state-audit` includes `external-resource-audit` for `v8.0.0` and
  later.
- Mission Control projects external resource readiness from canonical artifacts.
- The self-hosting repo contains a bounded read-only external resource example
  linked to `TASK-100`.

## Boundaries

- v8.0 does not implement autonomous external provider execution.
- v8.0 does not manage credentials, secrets, billing, deployment, or production
  writes.
- v8.0 does not prove semantic correctness of external output.
- v8.0 does not prove operator acceptance of real external workflows.

## Release Gates

1. `external-resource-audit` passes.
2. `release-state-audit` includes and passes `external-resource-audit`.
3. Mission Control schema includes external resource projection.
4. TASK-100 has Council review, work readiness, context integrity, execution
   packet, multi-actor, parallel-lane, requirement coverage, session export,
   and Archmap impact evidence.
5. `organization-verify`, command routing, CLI help benchmark, focused runtime
   tests, `npm test`, and `npm run smoke` pass.

## Next Frontier

After v8.0, the next frontier should move from local external resource
governance to provider adapter governance: approval policies, freshness checks,
safe read adapters, and explicit external-write escalation.
