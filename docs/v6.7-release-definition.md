# AOF v6.7 Release Definition

## Release

- version: `6.7.0`
- tag: `v6.7.0`
- theme: Verifiable Governance
- parent tasks: `TASK-072`, `TASK-073`, `TASK-074`, `TASK-075`, `TASK-076`

## Release Claim

`v6.7.0` makes release governance machine-verifiable.

The bounded claim is:

> AOF can reject release sign-off when implementation-grade work lacks an Archmap impact record, Council review provenance, or independent evidence category, and `release-state-audit` now consumes those gates before a release can be treated as ready.

## Required Outcomes

- `archmap-impact-audit` verifies implementation-grade tasks at or after `TASK-071` have resolvable Archmap impact records and non-pending Council disposition.
- `review-provenance-audit` verifies done work has Council review evidence with source task and parent session provenance.
- `evidence-independence-audit` verifies done work is not backed only by maker-authored or self-attested evidence.
- `release-state-audit` consumes all three governance audits as release sign-off gates.
- `cli-help-benchmark` fails if any supported command lacks AI-readable inputs or outputs.
- Mission Control / visibility no longer reports stale Archmap latest work when a later task has a higher task id.
- `TASK-076` closes the release with version, manifest, docs, checklist, and tests aligned to `6.7.0`.

## Authority Boundary

The release does not claim:

- semantic correctness of every review judgment
- market validation or product value validation
- that independent evidence proves truth by itself
- full external QIF verifier execution
- Node.js 25+ support

The release does claim:

- release sign-off now includes machine-checkable governance gates
- review provenance and evidence independence are first-class release readiness inputs
- command help completeness is enforced by a benchmark rather than assumed
- Archmap impact governance is checked automatically before release

## Gates

1. Archmap impact gate: `archmap-impact-audit` passes for `TASK-071` onward.
2. Review provenance gate: `review-provenance-audit` passes for done work at `TASK-071` onward.
3. Evidence independence gate: `evidence-independence-audit` passes for done work at `TASK-071` onward.
4. Release integration gate: `release-state-audit` includes all three governance audits and fails if any fail.
5. Command help gate: `cli-help-benchmark` fails missing help inputs or outputs.
6. Release surface gate: package, bootstrap, active release manifest, organization release contract, README, quickstart, checklist, and notes align to `6.7.0`.
7. Verification gate: organization verification, command routing, release-state audit, decision verification, full tests, and smoke validation pass.
