# AOF v8.4.0 Release Notes

## Summary

`v8.4.0` ships Externalized Runtime Operator Safety Proof.

The release adds a Mission Control safety surface for externalized runtime work.
Operators can now see whether external runtime advancement is safe, blocked,
approval-gated, stale, or not proven without opening raw audit JSON first.

## Added

- `external_runtime_safety_projection` in Mission Control export.
- Mission Control panel for permission, approval, provenance, freshness, risk,
  governance action, and not-proven boundaries.
- Schema coverage for externalized runtime safety states and checks.
- Focused regression tests for the safety projection.
- TASK-104 runtime evidence chain for v8.4 release closure.

## Preserved Boundaries

- This release does not authorize autonomous external writes.
- This release does not add external provider execution.
- This release does not prove semantic correctness of external tool output.
- This release does not replace human or governance approval.
