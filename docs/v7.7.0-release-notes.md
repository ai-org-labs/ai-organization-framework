# AOF v7.7.0 Release Notes

## Theme

Adoption-Grade v7 Runtime.

## What Changed

`v7.7.0` makes adoption proof executable. AOF no longer treats examples, docs, or generated artifact count as enough to claim adoption readiness. The release requires a fresh managed project to produce a bounded v7 runtime evidence chain.

## Runtime Changes

- Extended `adoption-proof-benchmark` with AP-007 through AP-011.
- Added v7 runtime evidence generation to the adoption benchmark:
  - work readiness
  - context integrity
  - work execution packet
  - requirement coverage
  - provider-neutral session export
  - release-ready boundary
- Added adoption proof to `release-state-audit` for `7.7.0` and later.
- Updated the adoption proof schema to require AP-001 through AP-011.

## Governance Boundary

This release proves structural/runtime adoption readiness only.

It does not prove:

- external market adoption
- operator acceptance by a real new adopter
- semantic quality of the adopter's idea
- provider-specific hosted onboarding automation
- Node.js 25+ support
