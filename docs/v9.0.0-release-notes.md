# AOF v9.0.0 Release Notes

## Summary

`v9.0.0` adds External Runtime Operator Acceptance and Safety Drill.

The release closes the gap after `v8.9.0`: outcome evidence and learning-loop updates are useful, but they are not enough unless an operator can inspect the full external runtime chain and record a bounded go/no-go style decision.

## Added

- `operator-acceptance-drill-record`
- `operator-acceptance-drill-audit`
- operator acceptance drill schemas
- CLI help and command registry entries
- Mission Control operator acceptance projection
- release-state audit gate for `v9.0.0` and later
- self-hosting drill evidence for `TASK-110`

## Verified Chain

The committed self-hosting drill links:

- provider execution approval
- provider execution reproduction
- provider rollback proof
- provider outcome evidence
- provider learning-loop update
- Mission Control packet
- operator decision and safety boundary

## Not Proven

This release does not prove production external execution safety, live provider behavior, credential safety, billing safety, semantic truth, market truth, independent adoption, or real rollback execution.

## Next Frontier

The next frontier should move toward a controlled provider execution candidate only if the operator acceptance boundary remains explicit and production authority is separately approved.
