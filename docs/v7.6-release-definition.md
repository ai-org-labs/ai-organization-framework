# AOF v7.6 Release Definition

## Release

- version: `7.6.0`
- tag: `v7.6.0`
- theme: Provider-Neutral Session Export
- release task: `TASK-095`

## Release Claim

`v7.6.0` makes AI work session evidence portable by requiring a provider-neutral session export before portability or release-ready reconstruction can be claimed.

The bounded claim is:

> AOF can record and audit a provider-neutral Session Export that links prompt, response, tool-call, artifact-write, verification, blocker, and stop-condition event summaries to task refs, requirement refs, test evidence refs, artifact refs, risk candidates, decision candidates, release-ready evidence, redaction boundaries, source-of-truth boundaries, and not-proven limits.

## Required Outcomes

- `session-export-record` writes schema-valid provider-neutral session export artifacts.
- `session-export-audit` verifies export presence, task linkage, accepted status, required event types, source session ref, task/requirement/test/artifact/release-ready refs, source-of-truth boundary, redaction boundary, release-ready boundary, and not-proven boundary.
- `release-state-audit` includes `session-export-audit` for `7.6.0` and later releases.
- TASK-095 has work readiness, context integrity, agent session, actor roster, actor output handoff, Council review, Work Execution Packet, Multi-Actor Pilot, Parallel Lane Pilot, Requirement Coverage, Archmap impact, independent evidence, and Session Export evidence.
- Provider streams are treated as input evidence; the AOF export package is the canonical portable runtime evidence surface.

## Authority Boundary

The release does not claim:

- semantic correctness of prompts or responses
- universal privacy safety
- provider API integration
- real-time stream ingestion
- external SIEM integration
- external adoption
- Node.js 25+ support

The release does claim:

- session evidence can be normalized into a provider-neutral package
- export readiness can require redaction and source-of-truth boundaries
- AI work can be linked to tasks, requirements, tests, risks, decisions, and release-ready evidence
- release readiness can reject missing provider-neutral export evidence

## Gates

1. Contract gate: export and audit schemas exist.
2. Writer gate: `session-export-record` creates schema-valid export artifacts.
3. Audit gate: `session-export-audit` fails missing exports and passes TASK-095 release evidence.
4. Release integration gate: `release-state-audit` includes Session Export audit for `7.6.0` and later.
5. Governance gate: TASK-095 has upstream work readiness, context integrity, agent session, Council review, Work Execution Packet, Multi-Actor Pilot, Parallel Lane Pilot, Requirement Coverage, Archmap impact, and independent evidence.
6. Boundary gate: export is not described as semantic truth, privacy proof, provider integration, real-time ingestion, or external adoption.
7. Verification gate: focused runtime tests, governance audits, release-state audit, organization verification, command routing, CLI help benchmark, npm test, and smoke validation pass.
