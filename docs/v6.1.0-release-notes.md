# AOF v6.1.0 Release Notes

## Summary

`v6.1.0` is the Work Governance Integration release.

It follows `v6.0.0` Public Runtime Readiness and expands AOF from a software-task-oriented public runtime into a cross-domain Human + AI + Tool Work Governance baseline.

## What Changed Since v6.0.0

- Added `docs/v6.1-work-governance-integration-plan.md`.
- Added `docs/v6.1-task-breakdown.md`.
- Added `docs/v6.1-work-governance-examples.md`.
- Added Work Governance as a core model concept.
- Added Work Governance schemas for work item goal, actor composition, Council-ready output, Go / No-Go visualization, Operational Map change log, Context Pack, and External Reference.
- Added software and manufacturing fixture chains.
- Added narrow CLI writers:
  - `work-item-goal`
  - `actor-compose`
  - `council-ready-output`
  - `go-no-go-visualization`
  - `operational-map-change-log`
  - `context-pack`
- Added `work-governance-benchmark`.
- Added Work Governance projection data to Mission Control.
- Added Work Governance Chain rendering to `visibility-serve`.
- Fixed runtime situation assessment so assigned frontier tasks and explicit task IDs are not ignored.

## Known Boundaries

- Work Governance is not yet fully autonomous.
- Bidirectional external-system sync is out of scope.
- Mission Control is still a read-only projection of runtime artifacts.
- Fixture pass does not prove real-world software release truth or manufacturing quality truth.
- Some originally proposed audit surfaces remain consolidated under `work-governance-benchmark` rather than productized as separate commands.

## Recommended Next Frontier

After `v6.1.0`, the next frontier should be selected by runtime review. The strongest candidates are:

- adoption proof: fresh project install, init, and first governed work item
- Mission Control redesign using Work Governance artifacts as the grounding layer
- broader operational-map audit and external-reference audit productization
- governed multi-actor orchestration beyond a single parent runtime
