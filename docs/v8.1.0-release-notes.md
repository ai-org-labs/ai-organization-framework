# AOF v8.1.0 Release Notes

## Summary

`v8.1.0` turns provider adapters into governed runtime contracts.

`v8.0.0` made external runtime resources and their uses auditable. `v8.1.0`
adds the adapter layer between AOF and those resources, so adapter readiness can
be checked before a provider path is treated as safe.

## Added

- `provider-adapter-record`
- `provider-adapter-audit`
- `schemas/aof-provider-adapter-record.schema.json`
- `schemas/aof-provider-adapter-audit.schema.json`
- provider adapter QIF/help mapping
- `provider_adapter_projection` in Mission Control
- `provider-adapter-audit` release gate for `8.1.0` and later
- self-hosting provider adapter policy and audit evidence

## Verified Boundary

AOF can now reject provider adapters that have missing refs, missing authority
boundaries, missing freshness checks, missing approval policy, write-capable
modes without escalation, or dangerous mode marked as `ready`.

## Not Proven

This release does not prove external provider semantic truth, operator
acceptance, credential safety, billing safety, production write safety, or
autonomous provider execution.
