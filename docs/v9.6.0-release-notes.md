# AOF v9.6.0 Release Notes

## What You Can Do Now

You can require a provider cost and quota boundary before a provider-backed action advances.

Before v9.6, AOF could show a controlled execution candidate and its incident recovery path, but cost, token, call, retry, quota, rate-limit, and billing limits were still scattered across boundary text.

After v9.6, AOF can record and audit one explicit cost/quota boundary: who owns the budget, what the cost ceiling is, how many tokens/calls/retries are allowed, what quota and rate limits apply, what happens on overage, and whether execution must be blocked or escalated.

## Capability Delta

NEW:

- `PCAP-PROVIDER-COST-QUOTA`: AOF can bind provider-backed execution to budget and quota evidence.

UPDATED:

- `PCAP-PROVIDER-EXECUTION`: Provider execution candidates now have incident recovery plus cost/quota gates before production go/no-go work can advance.

REMOVED:

- None.

## Capability Matrix

See [docs/aof-capability-matrix.md](aof-capability-matrix.md).

In thirty seconds:

> v9.5 showed what happens if a provider-backed action fails. v9.6 shows whether the action is allowed to spend tokens, calls, quota, or money at all.

## Value Evidence

Before:

- Operators had to infer cost and quota safety from approval, production-boundary, or provider notes.

After:

- Operators can inspect one cost/quota record that declares budget owner, budget period, cost ceiling, token/call/retry caps, quota/rate limits, billing boundary, overage policy, and stop conditions.

Reduced work:

- The operator no longer has to reconstruct provider spend risk from scattered artifacts before deciding whether provider execution may advance.

Five-minute demo:

```bash
node ./src/cli.js provider-cost-quota-boundary-audit --project .
node ./src/cli.js release-state-audit --project .
```

## Release Gate

`release-state-audit` includes `provider-cost-quota-boundary-audit` for `9.6.0` and later.

## Not Proven

v9.6 does not prove actual provider billing correctness, actual quota enforcement, production execution safety, provider semantic correctness, third-party adoption, or market value.

It proves that provider-backed execution must expose bounded cost and quota evidence before the external runtime advances.
