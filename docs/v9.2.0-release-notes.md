# AOF v9.2.0 Release Notes

## What You Can Do Now

You can ask AOF whether real provider execution is even eligible before it happens.

Before v9.2, AOF had approval, reproduction, rollback, outcome, learning, operator acceptance, and product-value evidence, but those signals were not bundled into one explicit production-boundary candidate.

After v9.2, AOF can produce and audit a single boundary packet that says:

- what upstream execution evidence exists
- what credential and budget boundaries are required
- how approval can be revoked
- what rollback, monitoring, and incident handling must exist
- whether a human has made an explicit go/no-go decision
- whether the release value is understandable
- whether production execution is still blocked

## New Commands

- `provider-production-boundary-record`
- `provider-production-boundary-audit`

## New Schemas

- `schemas/aof-provider-production-boundary-record.schema.json`
- `schemas/aof-provider-production-boundary-audit.schema.json`

## Mission Control

Mission Control now includes `provider_production_boundary_projection`, a read-only projection from the production-boundary audit artifact.

The projection must not be read as production safety. It is pre-production boundary evidence only.

## Release Gate

`release-state-audit` now includes `provider-production-boundary-audit` for `9.2.0` and later.

## Not Proven

This release does not prove:

- live provider execution
- production safety
- provider semantic truth
- credential safety
- billing safety
- actual rollback execution
- market adoption
