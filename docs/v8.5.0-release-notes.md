# AOF v8.5.0 Release Notes

## Summary

`v8.5.0` ships Provider Adapter Execution Pilot Boundary.

The release adds governed provider adapter pilot records, a pilot audit command,
Mission Control pilot readiness visibility, and release-state gating. Provider
adapter pilots are now represented as bounded experiments with dry-run/default-
deny behavior, approval state, provenance, redaction, rollback, verification,
stop conditions, and not-proven boundaries.

## Added

- `provider-adapter-pilot-record` CLI command.
- `provider-adapter-pilot-audit` CLI command.
- Provider adapter pilot record and audit schemas.
- Mission Control provider adapter pilot projection.
- Mission Control provider adapter pilot readiness projection.
- `release-state-audit` gate for provider adapter pilot evidence from v8.5.
- TASK-105 self-hosting provider adapter pilot evidence.

## Preserved Boundaries

- This release does not execute production external writes.
- This release does not grant autonomous provider execution authority.
- This release does not prove provider semantic correctness.
- This release does not manage secrets, billing, deployment, or irreversible
  operations.

## Verification

- `provider-adapter-pilot-audit` passes for the committed self-hosting dry-run
  pilot.
- Focused runtime tests pass.
- Final release verification is recorded in `docs/v8.5-release-checklist.md`
  and `.aof/context/active/release-state-audit.json`.
