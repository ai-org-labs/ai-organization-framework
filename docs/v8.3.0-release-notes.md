# AOF v8.3.0 Release Notes

## Summary

`v8.3.0` adds Mission Control Operator Acceptance Evidence.

The release moves operator validation from a backend audit fact into a
human-legible Mission Control state. Operators can now see whether feedback was
accepted, rejected, unclear, not reproduced, blocked, or missing, and can click
through to the governing audit and evidence refs.

## Added

- Expanded `operator-validation-audit` public records with feedback summary,
  work item ref, Mission Control ref, evidence refs, blocking reason, and
  not-proven boundary.
- Expanded Mission Control schema for operator validation evidence projection.
- Operator Acceptance Evidence panel in Mission Control.
- Runtime tests that require Mission Control projection to carry human-readable
  operator validation evidence.

## Changed

- Mission Control now treats operator acceptance as a first-class visibility
  surface rather than a count-only projection.
- The v8.3 release surface is bounded to acceptance evidence usability, not
  broader external execution authority.

## Boundary

Operator acceptance evidence is governed feedback visibility. It is not market
truth, semantic correctness, product value proof, provider correctness, or
permission to perform external writes.

## Next

The next frontier is `v8.4`: Externalized Runtime Operator Safety Proof.
