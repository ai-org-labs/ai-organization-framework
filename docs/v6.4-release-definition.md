# AOF v6.4 Release Definition

## Release

- version: `6.4.0`
- tag: `v6.4.0`
- theme: Pre-Implementation Quality Gate and QIF-Governed Explanation
- parent task: `TASK-069`

## Release Claim

`v6.4.0` turns the recent AOF/QIF retrospectives into a release-grade operating direction.

The bounded claim is:

> AOF must not use AOF/QIF only as post-implementation evidence. For product, UI, data-boundary, public-path, or user-facing work, AOF must force pre-implementation problem/assumption review, negative acceptance, evidence grading, Guardian done-before questions, and QIF living-ledger updates.

## Required Outcomes

- v6.4 direction document exists: `docs/v6.4-pre-implementation-quality-gate.md`.
- AOF records that solution-shaped tasks require problem-before-solution review.
- Product assumption review is required when sample/new/template/seed/project-loading/editor/projection/governance boundaries change.
- Negative acceptance is first-class release-quality evidence.
- Corrected assumptions can be recorded without erasing history.
- User-visible claims require user-visible E2E evidence.
- Code reading and symmetry reasoning are explicitly low-independence evidence for UI/public behavior claims.
- Guardian done-before questions cover adjacent paths, dependent surfaces, and public/irreversible risk.
- Visual/browser work requires rubric-based observation rather than screenshot existence.
- QIF is treated as a living ledger updated by bugs, corrections, contradicted assumptions, confidence changes, and residual risk.

## Authority Boundary

The release does not claim:

- full automation of the new gates
- semantic truth of any product decision
- executable QIF v0.3 verifier support
- Mission Control UX redesign
- complete prevention of bad implementation choices

The release does claim:

- the next AOF operating direction is no longer merely benchmark explanation
- v6.4 explicitly targets pre-implementation stopping power
- quality claims must distinguish evidence independence and user-visible E2E observation
- done claims must include adjacent/downstream/public-path checks when relevant

2. Direction artifact gate: v6.4 direction doc states pre-implementation gate, negative acceptance, corrected assumption, evidence grading, Guardian questions, visual rubric, and QIF ledger requirements.
