# AOF Capability Matrix

This matrix is the human-facing release map. It answers: what can a user do now?

| Capability | First Version | v7.x | v8.x | v9.0-v9.2 | v9.3 |
|---|---:|---|---|---|---|
| Traceability | v7.0 | Session/task/evidence chain | External refs begin | Provider chain evidence | Unchanged |
| Organization Memory | v7.1 | Context/reference integrity | External resource state | Release state and feedback chain | Unchanged |
| Governance | v6.4 | Work execution and multi-actor gates | Provider governance gates | Operator/product gates | Unchanged |
| Human Approval | v8.7 | Not present | Independent approval artifact | Production boundary candidate consumes it | Unchanged |
| External Resource Access | v8.0 | Not present | External resources and adapters | Provider execution evidence chain | Unchanged |
| Provider Execution | v8.5 | Not present | Pilot and approval bridge | Reproduction, rollback, outcome, learning, production boundary | Deferred after capability-first gate |
| Rollback | v8.8 | Not present | Rollback readiness proof | Production boundary consumes rollback | Unchanged |
| Learning | v8.9 | Not present | Outcome-to-learning update | Production boundary consumes learning | Unchanged |
| Product Value Comprehension | v9.1 | Not present | Not present | Product value evidence gate | Improved |
| Capability-First Release | v9.3 | Not present | Not present | Informal/partial | Formal model, delta, matrix, and audit |

## v9.3 Delta In 30 Seconds

Before v9.3, AOF could require product value evidence, but each release could still feel like another pile of records and audits.

After v9.3, release readiness requires a user-facing capability register, a capability matrix, a capability delta, and a release audit that checks whether the release starts from what a user can do.

## Not A Quality Claim

This matrix does not prove that every capability is valuable in the market.

It proves that AOF no longer lets release value be hidden behind implementation mechanism.
