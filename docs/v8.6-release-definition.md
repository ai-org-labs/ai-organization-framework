# AOF v8.6 Release Definition

## Theme

`v8.6` ships the Provider Execution Approval Bridge.

The release does not execute production provider writes. It defines the runtime contract that must exist before any real provider side effect can be treated as authorized.

## Release Scope

- Add `provider-execution-approval-record`.
- Add `provider-execution-approval-audit`.
- Add schema validation for approval bridge records and audit output.
- Add release-state audit coverage for `v8.6.0` and later.
- Project approval bridge state into Mission Control.
- Preserve the v8.5 boundary: provider adapter pilots remain dry-run/default-deny governance evidence unless explicitly approved.

## Required Approval Bridge Evidence

Every bounded external write preflight must include:

- linked provider adapter pilot
- linked provider adapter
- work item and session refs
- approval decision
- approved execution mode
- explicit external-write authorization flag
- human approval ref when external write is authorized
- execution scope
- allowed operations
- denied dangerous operations
- side-effect boundary
- redaction boundary
- rollback plan
- credential boundary
- budget boundary
- provenance refs
- verification refs
- stop conditions
- production execution status
- not-proven boundary

## Boundary

`v8.6` proves structural/runtime approval readiness only. It does not prove production provider execution safety, credential safety, billing safety, provider output correctness, market value, or autonomous external execution.

Production execution remains out of scope. A bridge may record `preflight_approved`; it must not be treated as proof that production execution happened or is safe.

## Release Gates

- `provider-execution-approval-audit` passes.
- `release-state-audit` includes the provider execution approval gate for `8.6.0`.
- Mission Control exposes provider execution approval projection.
- Negative tests fail missing human approval evidence for external-write authorization.
- Existing v8.5 provider adapter pilot gates remain green.
