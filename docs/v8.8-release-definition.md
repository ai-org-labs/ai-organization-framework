# AOF v8.8 Release Definition

## Theme

External Runtime Reproduction and Rollback Proof.

`v8.8.0` makes the post-approval boundary stricter: an approved external runtime action is not trusted until AOF can reconstruct the approved execution path and prove rollback readiness from canonical evidence.

## Required Runtime Contract

- Provider execution reproduction records bind approval, adapter, target operation, work item, session, input fingerprint, reconstructed steps, replay evidence, verification evidence, and not-proven boundary.
- Provider rollback proof records bind approval, reproduction proof, target operation, rollback operation, rollback evidence, verification evidence, stop conditions, and not-proven boundary.
- Audits must reject missing refs, target mismatches, adapter/work/session drift, blocked reproduction, unsupported rollback, and rollback-not-ready states.
- Release-state audit must include both v8.8 gates.

## Boundary

`v8.8.0` does not execute production external writes.

It proves only that the approved preflight path can be locally reconstructed and that rollback readiness is represented as auditable evidence before any future production execution claim is trusted.

Not proven:

- provider output correctness
- credential safety
- billing safety
- production execution safety
- real rollback execution
- market or operator value
