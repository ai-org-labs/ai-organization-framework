# AOF v7.3.0 Release Notes

`v7.3.0` is the Governed Multi-Actor Pilot release.

It follows `v7.2.0` Work Execution Packet. v7.2 made execution claims bounded. v7.3 adds the next missing layer: multi-actor claims must show Council presence, actor roster evidence, actor output handoffs, Council judgment, and packet closure.

## What Changed

- Added `multi-actor-pilot-record`.
- Added `multi-actor-pilot-audit`.
- Added schemas for Multi-Actor Pilot and Multi-Actor Pilot audit results.
- Integrated `multi-actor-pilot-audit` into `release-state-audit` for `7.3.0` and later releases.
- Added focused runtime tests proving missing pilot records fail and complete pilot records pass.
- Added TASK-092 release evidence for readiness, context integrity, session evidence, actor roster, actor handoffs, Council review, Work Execution Packet, Archmap impact, and pilot audit.

## Why It Matters

Before v7.3, AOF could bound execution through a packet, but it could still be unclear whether "multiple actors" meant actual governed handoff or one parent AI silently role-playing everyone.

After v7.3, AOF can explicitly show:

- which parent orchestrator ran the pilot
- whether Visionary, Builder, and Guardian were present as Council roles
- which actor roster was proposed
- which actor output handoffs exist
- which Council judgment accepted or blocked the result
- which Work Execution Packet closed the work
- what the pilot explicitly does not prove

## Boundary

`v7.3.0` proves structural/runtime multi-actor governance evidence.

It does not prove:

- autonomous workforce performance
- semantic correctness
- external worker adapter readiness
- parallel execution correctness
- external adoption
- provider-native session streaming

## Post-v7.3 Direction

The next v7.x step should add governed parallel lanes only after this single bounded pilot remains auditable.

Recommended sequence:

1. `v7.4`: Governed Parallel Lanes
2. `v7.5`: Requirements Coverage, Forecasting, And Organization Analytics
3. `v7.6`: Provider-Neutral Session Export
4. `v7.7`: Adoption-Grade v7 Runtime
