# AOF v9.1.0 Release Notes

## What Users Can Do Now

Users can ask "what changed in this release?" and inspect a capability-first answer backed by runtime evidence.

v9.1 adds a release gate for product value comprehension:

- what became possible
- what was manual or unclear before
- what concrete scenario changed
- what can be tried in five minutes
- what work or thinking is reduced
- whether understanding is present or must be escalated

## Why This Release Exists

User feedback after v9.0 showed that AOF was technically auditable but product value was not legible.

That feedback is now handled as a runtime outcome:

1. user feedback triggered a self-audit
2. self-audit opened `TASK-111`
3. `TASK-111` added product value evidence
4. release-state audit now gates v9.1+ on value comprehension evidence

## New Runtime Surface

- `product-value-evidence-record`
- `product-value-evidence-audit`
- `product_value_evidence_projection`
- `aof-product-value-evidence-record.schema.json`
- `aof-product-value-evidence-audit.schema.json`

## Product Benchmark

Before:

> The release explained records, audits, schemas, gates, and projections.

After:

> The release explains what a user can now do, how it differs from the previous state, and what evidence supports that claim.

Five-minute demo:

```bash
node ./src/cli.js product-value-evidence-audit --project .
```

## Not Proven

This release does not prove market adoption, broad user satisfaction, semantic correctness, or production provider safety.

It proves that AOF can make release value comprehension a governed runtime artifact and release gate.
