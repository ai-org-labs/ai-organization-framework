# AOF v8.5 Release Definition

## Theme

Provider Adapter Execution Pilot Boundary.

`v8.5.0` defines provider adapter execution pilots as governed experiments, not
hidden production automation. AOF may record and inspect a pilot only when the
pilot is bounded by dry-run/default-deny behavior, explicit approval state,
provenance, redaction, rollback, verification refs, stop conditions, and a
not-proven boundary.

## Required Outcomes

- Add a provider adapter pilot record contract.
- Add a provider adapter pilot audit command.
- Add CLI command routing and AI help for pilot record/audit.
- Add release-state gating for `v8.5.0` and later.
- Project pilot state and readiness into Mission Control.
- Keep v8.4 external runtime safety visibility compatible.

## Non-Goals

- This release does not execute production external writes.
- This release does not manage secrets, billing, deploys, or irreversible
  actions.
- This release does not prove provider semantic correctness.
- This release does not grant autonomous external execution authority.

## Acceptance Boundary

AOF may claim provider adapter pilot readiness only when pilot records resolve
to adapter, work item, session, provenance, and verification evidence; write-mode
pilots require explicit approval evidence; and production/billing/secret/deploy
or irreversible actions remain denied by default.

Pilot readiness is runtime governance evidence only. It is not production safety,
credential safety, billing safety, semantic truth, or permission for autonomous
external writes.
