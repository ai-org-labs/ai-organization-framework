# AOF v6.8 Release Definition

## Release

- version: `6.8.0`
- tag: `v6.8.0`
- theme: Executable Quality Ledger
- parent tasks: `TASK-077`, `TASK-078`, `TASK-079`

## Release Claim

`v6.8.0` makes quality-relevant evidence changes appendable and auditable.

The bounded claim is:

> AOF can record QIF-relevant evidence events in a canonical quality ledger, audit that ledger for traceability and governance escalation requirements, and prevent ledger presence from being mistaken for semantic truth.

## Required Outcomes

- `quality-ledger-record` writes schema-valid quality ledger events.
- `quality-ledger-audit` verifies committed ledger events have task refs, runtime refs, QIF refs, evidence refs, state transition intent, and governance escalation where needed.
- The audit rejects semantic truth claims made from ledger presence alone.
- The audit rejects unresolved task/runtime/evidence refs.
- The audit can represent corrected assumptions, contradicted claims, confidence changes, and operator feedback as event categories.
- The command registry, CLI reference, and help benchmark include the new ledger commands.
- `TASK-078` closes with Council review evidence and an auditable committed ledger event.
- `TASK-079` closes the release with version, manifest, docs, checklist, notes, and verification aligned to `6.8.0`.

## Authority Boundary

The release does not claim:

- automatic QIF verdict computation
- semantic truth validation
- market validation or product value validation
- external QIF verifier replacement
- that a ledger entry proves quality by itself
- Node.js 25+ support

The release does claim:

- quality-related changes can be recorded as structured runtime artifacts
- release/review quality claims can be checked for traceability gaps
- low-confidence, contradicted, or context-mismatched results can be routed to governance review
- AOF can distinguish activity evidence from quality evidence more safely than before

## Gates

1. Ledger schema gate: quality ledger event and audit schemas exist and are covered by tests.
2. Ledger writer gate: `quality-ledger-record` creates schema-valid events.
3. Ledger audit gate: `quality-ledger-audit` passes on committed events and fails missing traceability, unsupported semantic truth claims, missing escalation, and missing state transition intent.
4. Command surface gate: command registry, CLI reference, and AI help include the ledger commands.
5. Governance gate: `TASK-078` has Council review provenance and independent runtime/test evidence.
6. Release surface gate: package, active release manifest, organization release contract, README, quickstart, checklist, and notes align to `6.8.0`.
7. Verification gate: quality ledger audit, release-state audit, organization verification, command routing, CLI help benchmark, decision verification, focused tests, full tests, and smoke validation pass.
8. Release-state integration gate: `release-state-audit` includes `quality-ledger-audit` for `6.8.0` and later releases.
