# AOF v9.1 Release Definition

## Theme

Product Value Comprehension Gate.

## Capability Statement

After v9.1, AOF can treat "users do not understand what changed" as a governed runtime outcome.

The framework can record release value evidence, audit whether a release explains user capability instead of only internal artifacts, and project that understanding state into Mission Control.

## Before

Release quality could be claimed through records, audits, schemas, gates, tests, and dashboards even when users could not explain what became possible.

## After

A release claim must include product value evidence:

- capability-first explanation
- before/after state
- concrete work scenario
- five-minute demo
- work reduction or time-saving statement
- cognitive load removed
- capability matrix rows
- understanding outcome
- governance action when value is unclear
- not-proven boundary

## Required Runtime Surface

- `product-value-evidence-record`
- `product-value-evidence-audit`
- `product_value_evidence_projection` in Mission Control
- `release-state-audit` includes `product-value-evidence-audit` for `9.1.0` and later

## Product Benchmark

Scenario:

> A user asks what changed from v9.0 to v9.1.

Expected answer:

> AOF can now make release value itself auditable. It does not just add runtime internals; it requires a capability-first explanation, before/after comparison, five-minute demo, and understanding outcome.

## Release Gates

Release is valid only if:

- product value evidence exists
- the evidence links release, work item, Mission Control, and value evidence refs
- the release explains before/after capability
- the release includes at least one capability matrix row
- unclear or missing understanding escalates to governance
- Mission Control projects product value evidence
- `release-state-audit` passes with the new v9.1 gate

## Boundary

v9.1 does not prove market value, broad adoption, semantic truth, or production provider safety.

It proves that release value comprehension is now a governed artifact and release gate.
