# v8.1 Provider Adapter Policy

Provider adapters are governed access paths, not hidden execution shortcuts.

An adapter may be treated as release-ready only when:

- its provider and external runtime resource refs resolve
- read authority and write authority are separately bounded
- freshness is checked before the adapter output is used
- approval policy is explicit and resolvable
- side effects are declared
- every `local_write`, `external_write`, or `dangerous` mode requires escalation
- dangerous adapters are never marked `ready`
- not-proven limits are explicit

This policy does not prove external provider output correctness, credential safety,
billing safety, or operator acceptance. It only defines the local runtime boundary
for adapter governance.
